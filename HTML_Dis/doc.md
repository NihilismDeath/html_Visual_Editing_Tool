1. 项目背景与目标
背景：现有 Agent 生成 PPT 效率较低，且 PPT 格式难以灵活定制。HTML 作为通用网页格式，可被任何浏览器打开，适合快速制作和分享汇报材料。

目标：打造一个可视化、可拖拽的 HTML 编辑器，让用户像操作 PPT 一样编辑内容，最终导出独立的 HTML 汇报文件，无需额外软件即可演示。

2. 核心功能需求
功能模块	具体需求
幻灯片管理	添加、复制、删除幻灯片；缩略图切换；保留多页结构
元素编辑	支持文本、矩形、圆形三种基本元素；拖拽移动位置；拖拽右下角调整大小；双击文本直接修改内容
样式调整	选中元素后通过工具栏调整：字体、字号、颜色、背景色、加粗、斜体、下划线、删除线、对齐方式
导入原始 HTML	粘贴任意 HTML 代码，自动提取 <body> 中的文本块和内联样式，生成可编辑的文本元素
导出汇报文件	将所有幻灯片数据打包为一个独立 HTML 文件，内含完整样式和脚本，可直接在浏览器中播放
3. 设计思路（技术方案）
3.1 数据模型
幻灯片（Slide）：包含唯一 ID 和元素列表。

元素（Element）：包含 ID、类型（text/shape）、形状（rect/circle，仅 shape 类型）、位置（x, y）、尺寸（width, height）、内容（content）以及样式对象（style）。

样式对象：存储 fontSize、fontWeight、color、backgroundColor 等 CSS 属性，与 DOM 样式一一映射。

全局状态：维护幻灯片数组、当前索引、选中元素 ID、自增 ID 计数器。

3.2 渲染机制
采用 “数据驱动” 方式：每次修改数据后，调用 renderAll() 重新渲染缩略图和幻灯片。

缩略图预览：按比例缩放每个元素的位置和字号，模拟全貌。

幻灯片渲染：为每个元素创建绝对定位的 DIV，应用样式；文本元素使用 contentEditable 实现双击编辑；形状元素用 flex 居中显示内容。

3.3 交互实现
拖拽移动：监听 mousedown 记录起始坐标，结合容器缩放比例计算位移量，更新数据后重新渲染。

缩放调整：使用右下角控制点（蓝色圆点），拖拽时根据鼠标移动计算新的宽高。

选择与样式同步：点击元素设置 selectedElementId，高亮边框，并同步工具栏状态（字体、颜色等）。

右键菜单：提供复制、删除、图层上移/下移等快捷操作。

键盘快捷键：Delete/Backspace 删除元素，Ctrl+D 复制元素，Escape 取消选中。

3.4 导入原始 HTML
使用 DOMParser 解析用户粘贴的 HTML 字符串，遍历 <body> 的直接子元素。

忽略 <script>、<style> 等不可见标签，提取文本内容和内联样式（fontSize、color、backgroundColor、textAlign 等）。

为每个文本块生成一个新的文本元素，按顺序垂直排列（超出高度换列），并添加到当前幻灯片。

3.5 导出独立 HTML
序列化当前 data 对象为 JSON。

构建新的 HTML 页面，包含与编辑器相同的样式表，但仅保留只读预览模式（缩略图 + 幻灯片展示）。

内嵌 JSON 数据，并通过 JavaScript 渲染到页面中，实现“一页即可播放”的演示文件。

4. 使用流程（用户视角）
打开编辑器，看到默认示例幻灯片。

可执行以下操作：

添加元素：点击“文本/矩形/圆形”按钮。

编辑内容：双击文本直接输入。

调整样式：选中元素后使用工具栏。

管理页面：左侧缩略图切换、添加、复制、删除。

导入 HTML：点击“导入HTML”，粘贴代码，自动生成元素。

导出：点击“导出HTML”下载汇报文件。

导出的 HTML 可直接在浏览器中打开，供汇报展示。

5. 技术栈
纯原生 JavaScript（无第三方库）：轻量、易维护。

CSS3：Flex/Grid 布局，绝对定位，过渡动画，自定义滚动条。

DOM API：用于解析、渲染、事件绑定。

6. 未来扩展方向
支持更多元素类型（图片、图表、表格）。

增加动画过渡效果（元素入场、页面切换）。

实现撤销/重做（历史记录）。

支持从本地文件导入/导出 JSON 数据。

响应式优化以适应移动端编辑。

7. 总结
本工具实现了从 HTML 到可视化编辑再到独立汇报文件的完整闭环，降低了制作演示内容的技术门槛，特别适合快速将 AI 生成的 HTML 内容转化为可交互的 PPT 风格展示。数据驱动、实时渲染、多页管理、导入导出等核心功能确保了编辑效率和交付质量。
